## 📋 ドキュメント管理原則

### 更新責任
- **高頻度更新**: 現在の状況、実装状況
- **中頻度更新**: 技術仕様、設計ガイド
- **低頻度更新**: 開発履歴、将来計画

### 品質基準
- **簡潔性**: 必要な情報を簡潔に記述
- **最新性**: 実装変更と同期して更新
- **分類**: 適切なディレクトリに配置

### アーカイブ方針
- **アクティブ**: 現在参照されるドキュメント
- **アーカイブ**: 履歴として保存される過去のドキュメント
- **参考資料**: 将来の検討材料として保存
