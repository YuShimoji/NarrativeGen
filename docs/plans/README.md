# plans ディレクトリガイド

このフォルダは、直近の開発計画を短期・中期・長期に分けて管理するためのものです。

## 参照順序

1. `docs/plans/SHORT_TERM_TASKS.md` (1-2週間)
2. `docs/plans/MID_TERM_TASKS.md` (1-2ヶ月)
3. `docs/plans/LONG_TERM_TASKS.md` (3ヶ月以降)

## 運用ルール

- 実装着手前に、対象タスクの `Status` / `Branch` / `Depends` を確認する。
- 実装が確定した要件は、必要に応じて `docs/tasks/TASK_XXX_*.md` に昇格させる。
- 実績は `docs/reports/REPORT_TASK_XXX_*.md` へ記録する。

## SSOT の扱い

- 進行中の計画: この `docs/plans/` 配下
- 実装仕様/作業指示: `docs/tasks/`
- 実施結果: `docs/reports/`
