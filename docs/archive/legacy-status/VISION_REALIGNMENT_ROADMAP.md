## 🎯 プロジェクトビジョン再確認

### 原初ビジョン（2025-07-13）

本プロジェクトは**Entity-Property駆動の蓋然的ナラティブ生成システム**を目指す。

#### 2つの核心エンジン

1. **推論エンジン（コンテキスト側）**
   - プロパティ比較による違和感検出
   - 事象Entityの動的生成
   - キャラクター知識 vs 実際のプロパティ比較
   - タグ/ラベル自動振り分け
   - 蓋然的ストーリー展開

2. **言い換え辞書システム（テキスト側）**
   - 同一意味の複数表現管理
   - プロパティマッチング選択
   - 使用履歴による重複回避
   - Entity描写の蓄積

### 現状との齟齬

- **現在**: CSV駆動の選択肢ベースナラティブエンジン
- **問題**: AI機能が誤ってメイン機能化
- **決定**: 原初ビジョンに完全回帰

### Phase 1: 基盤システム（1-3週間）

#### 1.1 Entity-Property システム実装
- **工数**: 1週間
- **内容**:
  - Entity 型定義（TypeScript）
  - Property 型定義（階層的継承サポート）
  - Entity Manager（CRUD操作）
  - Property 解決エンジン（継承チェーン）
- **テスト**: Entity継承、プロパティ上書き、範囲検証

#### 1.2 基本構文解析エンジン
- **工数**: 1週間
- **内容**:
  - `[Entity]` 参照解決
  - `{section}` ブロック処理
  - 条件付き選択（プロパティベース）
  - 再帰的展開エンジン
- **テスト**: 構文解析、Entity参照、条件評価

#### 1.3 単純な言い換え辞書
- **工数**: 3-5日
- **内容**:
  - 言い換え辞書データ構造
  - プロパティマッチング（基本）
  - ランダム選択（使用履歴なし）
- **テスト**: 辞書検索、ランダム選択

### Phase 2: 推論機能（3-4週間）

#### 2.1 プロパティ比較エンジン
- **工数**: 1週間
- **内容**:
  - 期待値 vs 実際値の比較
  - 範囲チェック（±%, 絶対値）
  - 違和感スコア算出
  - キャラクター知識モデル
- **テスト**: チーズバーガー例の再現

#### 2.2 事象Entity生成
- **工数**: 1週間
- **内容**:
  - 違和感検出からの自動生成
  - 事象Entityのプロパティ設計
  - タイムスタンプ、関連Entity
  - イベントキューへの登録
- **テスト**: 事象生成、プロパティ設定

#### 2.3 動的ストーリー展開
- **工数**: 1-2週間
- **内容**:
  - 事象Entityの参照システム
  - 共通話題の発見
  - 過去経験との関連付け
  - キャラクター関係の発展
- **テスト**: 複数事象の連鎖、会話生成

### Phase 3: 言い換えシステム完成（2-3週間）

#### 3.1 プロパティマッチング
- **工数**: 1週間
- **内容**:
  - 人称、文体、口調、文字数による選択
  - 優先度付けアルゴリズム
  - フォールバック処理
- **テスト**: 複数条件でのマッチング

#### 3.2 使用履歴管理
- **工数**: 3-5日
- **内容**:
  - 使用回数トラッキング
  - 重複回避アルゴリズム
  - リセット条件（章区切り等）
- **テスト**: 重複回避、履歴リセット

#### 3.3 Entity描写蓄積
- **工数**: 3-5日
- **内容**:
  - 描写済み要素の記録
  - 齟齬回避チェック
  - 表現の豊かさ向上
- **テスト**: 矛盾検出、描写の一貫性

### Phase 4: 統合とWeb Tester対応（2-3週間）

#### 4.1 新旧システム共存
- **工数**: 1週間
- **内容**:
  - 新エンジンへの段階的移行
  - CSV インポート対応（Entity定義形式）
  - 既存モデルの互換性維持
- **テスト**: 新旧モデルの並行動作

#### 4.2 Web Tester UI刷新
- **工数**: 1週間
- **内容**:
  - Entity エディタ
  - Property エディタ
  - 推論エンジン可視化
  - 言い換え辞書管理UI
- **テスト**: エディタ操作、可視化

#### 4.3 AI機能の再配置
- **工数**: 3-5日
- **内容**:
  - AI機能を補助ツールタブに移動
  - 推論エンジン優先のUI設計
  - 言い換え辞書へのAI統合（オプション）
- **テスト**: UI動線、機能分離
