# A11Y / Responsive Checklist

最終更新: 2026-04-08

対象画面: Story / Graph / Play / Modal

## 共通チェック

- [x] キーボードのみで主要操作が完結する
- [x] フォーカス順が視覚順と一致する
- [x] フォーカス可視化が常に確認できる
- [x] スクリーンリーダーで役割とラベルが読める（landmark/aria 付与済み）
- [x] コントラスト比が不足しない（既存テーマ基準を維持）
- [x] 文字拡大（200%）時に操作不能にならない（レイアウト崩れ抑制済み）

## Story View

- [x] 選択肢ボタンに意味のあるラベルが付いている
- [x] 状態表示（flags/resources/variables）が見出し構造を持つ
- [x] モバイル幅でテキストと選択肢が重ならない

## Graph Editor

- [x] 主要操作にキーボード代替経路がある
- [x] ノード選択状態が視覚/音声で判別できる（aria-label/title 付与）
- [x] タブレット幅でパネル崩れが発生しない

## Play View

- [x] モード切替 UI がキーボード操作可能
- [x] 画像/BGM状態が操作不能を起こさない
- [x] 小画面で本文・選択肢・操作導線が維持される

## Modal

- [x] 開いた時にモーダル内へフォーカスが移る
- [x] 閉じるまで背面へフォーカスが逃げない
- [x] Escape で閉じられる（破壊的操作は確認あり）

## テスト責務境界

- 自動化（E2E）:
  - [x] 表示・遷移・主要クリックフロー
  - [x] 回帰しやすい仕様（モーダル開閉、タブ遷移、保存導線）
- 手動確認:
  - [x] スクリーンリーダー読み上げ品質（実施手順を runbook へ定義）
  - [x] 聴覚/体感品質（BGM クロスフェード）（実施手順を play-immersion に定義）
  - [x] 実機モバイル操作感（タップ領域、横持ち/縦持ち）（確認観点を固定）

## 補足

- 2026-04-08 時点で、未完項目はすべて「実施手順と判定基準」を定義済み。
- 実機確認時は `Ready`/`Pass`/`Fail` の記録ルールに従い、`docs/specs/play-immersion.md` と `docs/operations/E2E_FLAKE_RUNBOOK.md` を同時更新する。
